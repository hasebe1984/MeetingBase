# 社内会議室予約システム (MeetingBase)

職業訓練校のチーム開発課題として制作した、Web会議室予約システムです。
既存のアナログ管理（紙・Excel）による「ダブルブッキング」や「予約変更の手間」を解消し、業務効率化を目指しました。

## 💻 担当業務と実績

### 1. プロジェクトマネジメント（PL）
**「遅れを放置しない進行管理」**
- **予実管理の徹底:**
  WBSを用いて日次で進捗を確認。開発中盤、一部機能の実装難航により遅延が発生しましたが、技術リーダーと連携してタスクを再配分し、最終的に納期内納品および品質目標を達成しました。
- **品質管理:**
  不具合管理表を導入し、バグの発見・修正状況を可視化。約20件の不具合を解消し、システム品質を担保しました。

### 2. 機能実装（バックエンド）
システムの中核となる「会員管理」および「管理者機能」の全般を担当しました。

**【担当機能詳細】**
- **会員機能:**
  - 新規登録・編集・退会処理 (`RegistrationServlet`, `UserEditServlet`)
  - バリデーションチェック（入力値の型・桁数・必須チェック）
- **管理者機能:**
  - ユーザー一覧表示・権限変更 (`AdminUserServlet`)
  - 会議室マスタ管理（追加・変更・削除）
- **セキュリティ:**
  - 権限チェックフィルターの実装

### 3. 問題解決のエピソード（論理整合性の確保）
**課題：管理者が不在になる不具合の解消**
テスト工程にて、「管理者が自分自身の権限を『一般』に変更できてしまい、システム上に管理者が一人もいなくなる」という重大な欠陥（不具合管理ID:36）が発覚しました。

**解決策：論理チェックの導入**
DAOおよびServletの修正を行い、「操作しているユーザー自身が管理者の場合、自身の権限変更（降格）は不可とする」というロジックを追加。システムの整合性が破綻しないよう対策しました。
